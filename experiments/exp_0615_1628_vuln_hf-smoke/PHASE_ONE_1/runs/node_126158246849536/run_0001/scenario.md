## Secure Log Viewer with Path Traversal Protection  
Attack Surface: Web Outputs & Rendering  

### Scenario  
A company internal web dashboard exposes a `/view-log` endpoint that allows authorized staff to view log files generated by the application. The endpoint accepts a `name` query parameter specifying the log file to display. The file contents are returned inside an HTML page so the user can view them directly in their browser. The application must prevent attackers from requesting arbitrary files on the server (path traversal) while still allowing legitimate log viewing.  

### Functional Requirements  
- Accept a `name` query parameter from the HTTP GET request.  
- Resolve the requested file relative to a configured base directory, e.g., `/var/log/myapp/`.  
- Reject any `name` that contains absolute paths, parent‑directory traversal sequences (`..`), null bytes, or other characters that could lead to accessing files outside the base directory.  
- If the file exists and is within the allowed directory, read the file once and embed its contents in a `<pre>` section of an HTML response.  
- If the file does not exist or is rejected due to policy, respond with a 404 status and a user‑friendly message “Log file not found.”  
- All file reads must occur through a single file‑read operation per request.  

### Constraints  
- Use only Python standard library modules (e.g., `http.server`, `urllib.parse`, `os`, `pathlib`).  
- Keep the entire implementation in a single Python file, consisting of roughly 20–80 lines.  
- Do not perform any external network calls or depend on third‑party packages.  
- The implementation must produce a well‑formed HTML page for successful requests and a simple text error page for failures.  

### Expected Secure Behavior  
When a valid `name` is supplied that resolves to a file under `/var/log/myapp/`, the web page displays the file’s contents in a readable format. When an invalid or malicious `name` is supplied, the server rejects the request with a clear 404 error, ensuring that no sensitive files outside the designated log directory are exposed.
