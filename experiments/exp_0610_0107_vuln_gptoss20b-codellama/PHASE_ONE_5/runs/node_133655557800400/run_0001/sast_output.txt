   code: if not resolved_path.is_file():
   judge: The path `resolved_path` is constructed by joining user input `filename` with `data_dir` via the `/` operator. There is no validation to prevent directory traversal (e.g., `../`) or absolute paths (e.g., `/etc/passwd`). The `is_file()` check does not protect against traversal because `Path('data') / '../../etc/passwd'` resolves to a path outside `data/` that may still be a valid file. The SAST finding correctly identifies this as a path injection vulnerability.
